# Tech Specs

A Tech Spec defines the **implementation contract** for an approved proposal — Clean Architecture layer breakdown, Firestore schema, Riverpod providers, acceptance criteria, and test plan. No code is written until the spec is approved.

Use the [Tech Spec stencil](../stencils/tech-spec.md) to start a new one. Save specs to `tech-specs/` at the repo root on the `main` branch as `NNNN-slug.md` (e.g. `0001-auth-flow.md`). Number matches its proposal.

## Workflow

```
Proposal → approved → Spec (this page) → approved → Implementation → Review
```

Each spec must reference the proposal it expands.

# Tech Proposals

A Tech Proposal defines the **problem, proposed solution, and alternatives** for a non-trivial feature. It must be approved before a Tech Spec is written.

Use the [Tech Proposal stencil](../stencils/tech-proposal.md) to start a new one. Save proposals to `tech-proposals/` at the repo root on the `main` branch as `NNNN-slug.md` (e.g. `0001-auth-flow.md`).

## Workflow

```
Proposal (this page) → approved → Spec → approved → Implementation → Review
```

Skip proposals only for changes touching ≤ 2 files with no architectural impact.
